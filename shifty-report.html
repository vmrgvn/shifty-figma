Shifty — Отчёт по фичам

Wizard v1.0 · Июнь 2026 · Внутренний документ для QA

Что добавлено

Визард — 4 шага

  • Шаг 1 — Команда и роли: добавление/удаление сотрудников и ролей, назначение ролей сотрудникам
  • Шаг 2 — Расписание: режим (каждый день / по дням недели / цикл N+M), выбор рабочих дней
  • Шаг 3 — Смены: группы конфигураций с временными блоками, привязка ролей к сменам
  • Шаг 4 — Настройки: название расписания, перерывы, выходные, минимальный отдых между сменами

Панели сотрудника (открываются чипами на шаге 1)

  • Роли — назначение из глобального списка
  • Отсутствия — больничный, отпуск, разовое / еженедельное / ежемесячное отсутствие с опциональным временем
  • Найм / увольнение — даты начала и окончания работы
  • Время — предпочтительное / нежелательное рабочее время по расписанию
  • Команда — кто с кем хочет или не хочет работать в смену

Прогресс-бар

  • На десктопе завершённые шаги кликабельны — можно вернуться назад по клику
  • Мобайл — только визуальный индикатор

Диалог генерации

  • 5 секунд анимации загрузки → экран «готово»
  • Кнопка «Создать аккаунт» → переход на страницу логина
  • Одна кнопка «Скачать Excel-файл» (было две)

Навигация

  • Иконка 🏠 вверху — выход на главную страницу
  • Кнопка «← Название предыдущего шага» внизу — переход на шаг назад
  • Черновик автосохраняется в localStorage, восстанавливается при возврате

Локализация

  • 11 языков: ruenkkdefresitpttrzhja
  • Все панели сотрудника, шаги 2 и 3, диалог генерации — полностью переведены

Тёмная тема

  • Иконка calendar/clock picker теперь видна в dark-режиме (CSS invert-фильтр)

Десктоп vs Мобайл

Десктоп Мобайл
Шаг 1 (Команда) Двухколоночный лэйаут: список слева, панель справа Панель сотрудника открывается как bottom sheet
Прогресс-бар Кликабельные завершённые шаги Только отображение
Роли на мобайле В правой колонке постоянно Отдельная кнопка «Роли» → sheet
Шаги 2–4 Одноколоночный, max-width 700px Полная ширина

Валидации

Шаг 1

  • Нельзя продолжить без хотя бы одного сотрудника
  • Нельзя продолжить без хотя бы одной роли
  • Нельзя продолжить если у кого-то из сотрудников нет роли

Шаг 3 (Смены)

  • Нельзя продолжить без хотя бы одной смены
  • Время окончания смены должно быть позже начала

Шаг 4 (Настройки)

  • Время окончания перерыва должно быть позже начала

Панели сотрудника

  • Дата увольнения не может быть раньше даты найма
  • В отсутствиях: дата окончания ≥ дата начала
  • Время окончания отсутствия / предпочтения > время начала
  • При попытке сохранить без обязательных полей — показывается подсказка об ошибке

Что проверять тестерам

Основной сценарий

  • Пройти все 4 шага → «Составить расписание» → дождаться анимации → «Создать аккаунт» → должен открыться логин
  • Закрыть визард на шаге 3, открыть снова → данные должны восстановиться
  • На шаге 1 добавить сотрудника, назначить роль, открыть все 5 панелей по очереди → чипы должны подсветиться после ввода данных

Валидации

  • Добавить сотрудника без роли, нажать «Продолжить» → ошибка
  • Создать смену с временем 18:00–09:00 → ошибка
  • Дата увольнения раньше найма → ошибка
  • Отсутствие «Другое → Еженедельно», не выбрать ни одного дня → нажать «Добавить» → ошибка
  • Перерыв с временем окончания раньше начала → ошибка

Навигация

  • Кликнуть по завершённому шагу в прогресс-баре на десктопе → должен переключиться
  • Иконка домика вверху → на главную
  • Кнопка «← Шаг X» внизу → на предыдущий шаг (не на главную)

Локализация

  • Переключить язык в середине заполнения → всё должно переключиться включая панели и ошибки
  • Тёмная тема: иконка календаря/времени видна в date/time инпутах

Вывод по ресурсам

Больше всего итераций и токенов ушло на threading переводов через дерево компонентов — тип PanelCopy с ~35 полями на 11 языков, и его проброс через четыре уровня: Wizard → StepTeam → renderFeaturePanel → AbsencesPanel / TimePrefsPanel / ... Каждое звено требовало отдельного редактирования. Edit-инструмент несколько раз не мог найти строку из-за CJK-символов и специфичного выравнивания в genCopy — пришлось переключаться на Node.js-скрипты для замены. Второе по затратам — полная перепись WizardStep3 с добавлением S3Copy через 6 вложенных компонентов. Остальное — точечные правки, каждая по 1–2 итерации.
